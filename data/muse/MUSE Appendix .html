The settings to include in the configuration are: 
computer_space_limit = 100000000
command_require_creative = false
The 'default' settings seem actually to override the settings for a world. Each world's settings are in the world's `saves` as `.minecraft\saves\\serverconfig\computercraft-server.toml`. To get started, for convenience, use a cheat to set some game rules and give a command computer to the player: Open the command computer The `base` option is needed just once to provide what's needed to launch a GPS facility using `rover` on top of `porter`. The `player` pocket computer, `rover` and the `porter` each need to run `amuse`. This work is done locally. They each need an advanced (ender) modem. With the parts given to the player by `kit base`, do some assembly work using the player's crafting facility. Craft a pocket computer with an ender modem on top. Craft a turtle with an ender modem on either the left or right side. Then take the ender modem equipped turtle and craft a sword on the side opposite. (Once the turtle is in the game, you can add a pickaxe from player inventory to its inventory.) Attach an ender modem to the back of the command computer using a `sneak`. The rest of the given `base` inventory will be used to setup the GPS computers by using the `launch` command. To do that, we'll need to align Muse dead reckoning for a turtle, likely `rover`, with the Minecraft coordinate system. That's done with a `locate` command. We'll get to that in a bit. Muse provides support for the idea of a `site`. There may be several of these in a world. Each could have a set of workers: a `farmer`, a `logger`, and a `miner`. A `kit site` command puts the turtles for a `site` in player inventory with the needed modems and work tools. They'll need to be registered with their roles by `amuse` for Muse operations. While GPS coordinates need not be aligned with Minecraft coordinates, it helps in using some game facilities using if they are. Place `rover` on top of the `porter` and run `locate` from the `player` pocket computer. At this point in the setup, remote operations should be working. Once `locate` has run, place the remaining player inventory items in `rover` inventory and issue the `launch` command to setup GPS. When setting up a new site, you may want to keep site workers working while the player is elsewhere.
